# /commit — コミットメッセージ生成

## 手順

1. `git diff --staged` または `git diff HEAD` で変更内容を確認する
2. 変更の性質を以下の型で分類する:

| 型 | 用途 |
|---|---|
| `feat` | 新機能追加 |
| `fix` | バグ修正 |
| `refactor` | 機能変更なしのリファクタリング |
| `test` | テスト追加・修正 |
| `docs` | ドキュメントのみの変更 |
| `style` | フォーマット・命名のみの変更（ロジック無変更） |
| `chore` | ビルド・設定・依存関係の変更 |

3. コミットメッセージを生成する:
   ```
   <type>: <概要（日本語、50字以内）>

   （必要であれば本文: 変更の理由・影響範囲）
   ```

4. 生成したメッセージを提示し、確認を取ってから `git commit` を実行する

## 制約

- `git add -A` や `git add .` は使わず、変更ファイルを明示して staging する
- `.env` / credentials 等センシティブファイルを含めない
- 1コミット = 1つの論理的変更（複数の independent 変更は分割を提案）
