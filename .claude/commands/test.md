# /test — テスト実行と失敗分析

## 実行手順

1. 以下のコマンドでテストを実行する（引数指定があれば `-k` フィルタを追加）:
   ```bash
   python -m pytest tests -v --tb=short 2>&1
   ```

2. 結果を日本語で要約する:
   - 合格 / 失敗 / エラー の件数
   - 失敗したテストケース名と原因
   - 物理的に問題がある可能性（許容誤差超過など）か、コードのバグか判断する

3. 失敗がある場合:
   - 失敗したテストのソースを Read で確認する
   - 対応する実装コードを Read で確認する
   - 根本原因を特定して修正案を提示する（自動修正は確認を取ってから）

## 引数

- 引数なし: 全 65 件を実行
- クラス名: `python -m pytest tests -k "クラス名" -v`
- ファイルパス: そのファイルのみ実行

## 注意事項

- `np.allclose` / `np.isclose` で比較している物理量は許容誤差に注意
- `rl_load_solver.py` の差分離散時間更新は意図的な逐次処理（ベクトル化禁止）
- 偽陰性・偽陽性を避けるため `atol` / `rtol` を確認すること
