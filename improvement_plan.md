# 改善ロードマップ

## 1. 現在地

現行実装では、以下はすでに完了している。

- dt_actual を用いた時間刻み整合
- 助走区間を含む定常状態表示
- V_LL(rms) 入力と内部 peak 変換
- m_a 表示と過変調クランプ通知
- 線間電圧 FFT と THD 表示
- FFT 精度向上と電流スペクトル拡張
- 基本波オーバーレイと理論電流比較
- PWM 方式比較モード（Natural / Third Harmonic Injection / Min-Max Zero-Sequence）
- デッドタイムと固定導通電圧降下を含む簡易非理想インバータモデル
- RL ソルバの厳密離散化
- 学習シナリオガイド機能（IMPROVE-11: desktop UI + web UI）
- 条件比較・エクスポート機能（IMPROVE-12: desktop UI + web UI）

この文書では、これまでに実施した改善項目の記録を扱う。現時点で未実装の改善項目はない。

## 2. 優先順位の考え方

優先度は次の順で決める。

1. 理論的正確性を上げる改善
2. 教育効果を定量的に高める改善
3. 再現性、比較容易性、運用性を上げる改善

## 5. 詳細計画

### IMPROVE-11: 学習シナリオガイド機能

#### IMPROVE-11 の目的

現行 UI は自由度が高い一方で、初学者はどこを見るべきか分かりにくい。教育効果を上げるには、観察ポイントを自動で提示する導線が必要である。

#### IMPROVE-11 の実装内容

- プリセットボタンまたはシナリオ選択 UI を追加する
  - 線形変調限界
  - キャリア周波数の効果
  - L の平滑効果
  - 位相遅れ
  - 過変調とクランプ
- 選択中シナリオに応じて注目指標をハイライトする
- 利用手順書にシナリオ対応表を追加する

#### IMPROVE-11 の変更候補ファイル

- ui/visualizer.py
- docs/user_guide.md
- README.md

#### IMPROVE-11 の受け入れ条件

- 初学者が 5 分以内に主要観察点へ到達できる
- シナリオ切替でパラメータと説明が一貫して更新される
- 既存の自由操作モードを妨げない

### IMPROVE-12: 条件比較・エクスポート機能

#### IMPROVE-12 の目的

授業や自己学習では、変更前後の条件を並べて比較したり、結果をレポートへ貼り付けたりする需要がある。現状は画面を見て覚えるしかなく、再現性が弱い。

#### IMPROVE-12 の実装内容

- 現在パラメータを JSON または CSV で保存できるようにする
- PNG で波形画像を保存できるようにする
- ベースライン条件と比較条件を重畳または左右比較できるようにする
- 保存ファイルに主要指標 m_a, m_f, V1, THD を含める

#### IMPROVE-12 の変更候補ファイル

- ui/visualizer.py
- docs/user_guide.md
- README.md

#### IMPROVE-12 の受け入れ条件

- 同一条件を保存・再読込して同じ波形を再現できる
- 授業資料向けに画像と主要数値を出力できる
- 比較前後の条件差が UI 上で明示される

## 7. 進捗管理ルール

各改善項目は、着手前に以下を満たすこと。

- 対象物理モデルの前提を文書化する
- 既存テストへの影響を明示する
- README、利用手順書、.github 指示書の更新対象を列挙する

各改善項目は、完了時に以下を満たすこと。

- 少なくとも 1 件の自動テストを追加または更新する
- docs/user_guide.md の説明を同期する
- 変更が教育効果にどう効くかを短く記録する

## 8. 保留事項

以下は今後も慎重判断とする。

- SciPy を正式依存にするかどうか
- SVPWM を本プロジェクトの対象に含めるかどうか
- モータモデルまで範囲拡張するかどうか

ここは教育対象の深さと実装複雑度のトレードオフになるため、機能追加前に目的を再確認する。
